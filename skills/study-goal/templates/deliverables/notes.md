# 노트형 산출물 변형 생성 가이드

## 4가지 변형 생성 패턴

다음 스타일 중 4개로 변형:

1. **Cheat sheet** — 1페이지 요약, 핵심 공식/개념만 (~2-4시간)
2. **Concept map** — 개념 간 관계도 + 각 개념 1문단 설명 (~반나절)
3. **Deep notes** — 챕터별 상세 노트, 예제 + 본인 언어 재구성 (~1-2일)
4. **Q&A 형식** — 자문자답으로 정리, 빈틈 발견용 (~1일)

## 완료 체크 항목 작성 가이드

각 변형의 완료 항목에 다음을 포함:
- 특정 파일이 존재 (`notes.md` 등)
- 다루기로 한 토픽 N개가 모두 포함됨 (목록을 명시)
- 본인 언어로 재구성된 흔적 (단순 복붙 금지)
- 최소 한 개의 본인 예시 또는 비유
- (Q&A의 경우) 본인이 답을 못한 질문 1개 이상 (정직성 체크)

## 좋은 예 / 나쁜 예

**나쁨**: "트랜스포머에 대한 노트를 만든다"
**좋음**:
- [ ] `notes/transformer.md`에 다음 7개 토픽 모두 다룸: tokenization, embedding, self-attention, multi-head, position encoding, FFN, residual & layer norm
- [ ] 각 토픽마다 본인 언어 설명 + 본인 예시 1개
- [ ] 최소 1개 비유 (예: "attention = 검색어로 도서관에서 책 찾기")
- [ ] 마지막에 "아직 헷갈리는 것 3개" 섹션
