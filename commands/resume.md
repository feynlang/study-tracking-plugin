study-goal 스킬의 재개 흐름을 실행한다.

사용자 지정 slug: $ARGUMENTS

## 인자가 있을 때 ($ARGUMENTS 비어있지 않음)

1. `~/study-log/<slug>/artifact.md` 존재 확인
   - 없으면: 에러 메시지 + 비슷한 slug 후보 제안 + `/study-goal:list` 안내
2. 상태 확인
   - `🟢 완료`인 경우: "이 목표는 이미 완료된 상태예요. 다시 진행하려면 새 목표로 시작하거나 명시적으로 unfreeze 요청해주세요"
   - `🟡 진행 중` 또는 `🔴 일시중지`: 재개 가능
3. `.active` 갱신:
   - single-active 모드(추정): `.active` 내용을 이 slug 한 줄로 덮어쓰기
   - multi-active 모드(이미 여러 줄): 이 slug를 첫 줄로 이동 (없으면 추가)
4. 상태가 `🔴 일시중지`였으면 `🟡 진행 중`으로 변경
5. 사용자에게 현재 상태 요약 표시:
   - 토픽/콘텐츠 제목
   - 종료 조건
   - 산출물 요구사항 진행률 (M/N 항목 완료)
   - 마지막 작업 로그 시각 + 무엇을 했었는지
   - 가장 최근의 "막힌 점" (있으면)
6. artifact.md의 작업 로그 섹션에 재개 기록 append:
   ```
   ### YYYY-MM-DD HH:MM
   **재개**: 마지막 작업으로부터 <시간 차> 만에 재개
   ```
7. "어디서부터 이어갈까요?" 묻고 사용자의 다음 액션 대기

## 인자가 없을 때 ($ARGUMENTS 비어있음)

`~/study-log/`의 모든 디렉토리를 스캔해서 분류:
- 진행 중 목표 (`🟡` 상태): 후보
- 일시중지 목표 (`🔴` 상태): 후보
- 완료 목표 (`🟢` 상태): 제외

분기:
1. **후보 0개**: "재개할 목표가 없어요. `/study-goal:new`로 시작하세요"
2. **후보 1개**: 자동으로 그 목표 재개 (위 인자 있을 때 절차)
3. **후보 여러 개**: AskUserQuestion으로 목록 제시, 사용자 선택 후 재개
   - 각 옵션 라벨: `<제목> (진행률 M/N, 마지막 작업: N일 전)`
   - 활성 목표(`.active`에 있는 것)는 라벨에 `(활성)` 표시

## 자연어 트리거

이 명령은 다음 자연어 패턴에서도 동작해야 함:
- "X 이어서 할게", "X 다시 시작", "X 계속할게"
- "어제 하던 거", "아까 하던 거"
- "다시 RAG로 돌아가자"

이런 표현 감지 시 `/study-goal:resume [감지된 slug]`로 매핑.
