# 링크 모드: GitHub 레포 학습 산출물 변형 가이드

## 4가지 변형

1. **타이핑 재현** — 한 글자씩 따라 타이핑하며 주석 달기 (~1-3일)
2. **fork + 확장** — 의미 있는 기능 1개 추가 (예: 다른 데이터셋, 시각화) (~2-5일)
3. **scratch 재구현** — 원본 보지 않고 처음부터 작성 (~1주+)
4. **해부 노트** — 모든 함수/클래스에 "왜 이렇게 짰는지" 분석 노트 (~2-4일)

## 완료 체크 항목 작성 가이드

### 타이핑 재현
- [ ] 원본의 모든 핵심 파일 직접 타이핑 완료 (복붙 X)
- [ ] 각 함수/클래스에 본인 언어 주석 추가
- [ ] 원본과 동일하게 실행되어 동일한 결과 확인
- [ ] 막혔던 부분 / 새로 배운 점 회고 노트

### fork + 확장
- [ ] 원본 fork
- [ ] 추가할 기능 1개 명시 (예: "transformer block에 Flash Attention 적용")
- [ ] 기능 구현 + 작동 검증
- [ ] CHANGES.md에 변경사항 + 결과 비교

### scratch 재구현
- [ ] 원본 코드 안 보고 작성 (README/논문은 OK)
- [ ] 핵심 기능 N개 동일하게 작동
- [ ] 본인 코드 vs 원본 코드 차이점 분석 노트
- [ ] 막혔을 때만 원본 참조한 횟수 기록

### 해부 노트
- [ ] 주요 파일 N개에 대해 분석 문서
- [ ] 각 함수마다 "왜 이렇게 짰는가" 추론
- [ ] 의문점 N개 이상 (그리고 답을 찾으려 시도)
- [ ] 디자인 결정의 trade-off 분석 1편

## 변형 추천 가이드

레포 크기/복잡도에 따라:
- 작고 명확한 레포 (예: nanoGPT 같은 < 1000줄) → scratch 재구현 추천
- 중간 (1000-5000줄) → 타이핑 재현 또는 fork+확장
- 큰 레포 (5000+줄) → 해부 노트 (전체 재현은 비현실적)

수준에 따라:
- 초급 → 타이핑 재현 (안전, 학습 효과 높음)
- 중급 → fork+확장 또는 해부 노트
- 상급 → scratch 재구현
