---
description: 플러그인 소개와 활성 학습 목표 대시보드 (처음 사용 시 시작점)
---

study-goal 플러그인의 온보딩 + 현재 상태 대시보드.

이 명령은 두 가지를 한 번에 함:
1. 플러그인 사용법 안내 (특히 처음 쓰는 사용자에게)
2. 활성/일시중지/완료된 학습 목표 현황 표시

## 절차

### 1. 첫 사용 여부 판단

`~/study-log/` 디렉토리 존재 여부로 판단:
- **없음**: 처음 사용 → 풀 온보딩 (아래 §2-A)
- **있음**: 기존 사용자 → 짧은 인사 + 대시보드 (아래 §2-B)

### 2-A. 첫 사용자 온보딩

다음을 한 메시지로 보여줌 (간결하게):

```
👋 study-goal 플러그인에 오신 걸 환영해요!

이 플러그인은 **모호한 학습 의도를 반드시 산출물로 이어지게 강제**합니다.
"공부했다"라고 말하기 전에 손에 잡히는 결과물이 있어야 한다는 철학이에요.

## 워크플로 (한 번 보면 끝)

1. 학습 목표 던지기 ("X 공부하고 싶어" 또는 URL 공유)
2. 적응형 개념 체크로 현재 수준 측정
3. 종료 조건 4지선다 (얼마나 하면 끝?)
4. 산출물 형태 4지선다 (프로젝트/노트/문제풀이/설명자료)
5. 산출물 스펙 4지선다 (구체적으로 뭘 만들지)
6. artifact.md 자동 생성 + 작업 동안 자동 로깅

## 명령어

| 명령어 | 용도 |
|--------|------|
| `/study-goal:start` | (지금 보고 있는 화면) 온보딩 + 대시보드 |
| `/study-goal:new` | 새 학습 목표 시작 (일반 토픽) |
| `/study-goal:link <URL>` | 링크 기반 학습 (GitHub 레포 / 블로그 / 논문) |
| `/study-goal:list` | 진행 중인 모든 목표 조회 |
| `/study-goal:resume [slug]` | 일시중지된 목표 재개 |
| `/study-goal:pause [slug] [사유]` | 진행 중 목표 일시중지 |
| `/study-goal:check` | 산출물이 종료 조건 만족하는지 평가 |
| `/study-goal:sidebar [질문]` | 본 작업 중 떠오른 샛길 질문 처리 |

## 자연어로도 OK

명령어 안 외워도 됩니다. 다음과 같이 말해도 동작해요:
- "트랜스포머 공부하고 싶어" → 새 목표 시작
- "어제 하던 거 이어서" → 재개
- "잠깐 멈출게" → 일시중지
- "잠깐 질문 하나만" → 샛길 모드

## 시작해 볼까요?

지금 바로 시작하고 싶은 학습 주제가 있으세요? 토픽이든 URL이든 말씀해 주세요.
없으면 그냥 "둘러볼게"라고 하셔도 돼요.
```

이후 사용자 응답에 따라:
- 토픽/URL → Step 1로 이동
- "둘러볼게" → 더 자세한 예시 워크플로 1개 보여주고 대기

### 2-B. 기존 사용자 대시보드

`~/study-log/` 안의 모든 디렉토리 (메타 디렉토리 제외) 순회해서
각 artifact.md에서 추출:
- 토픽/콘텐츠 제목
- 모드 (일반 / 링크)
- 상태 (🟡 / 🔴 / 🟢)
- 진행률 (M/N 체크 항목)
- 마지막 작업 시각

`.active`와 비교해 활성 표시.

표 형식으로 출력 (3개 그룹):

```
👋 다시 오셨네요! 현재 학습 현황이에요.

## 🎯 활성 (.active에 있음)
| 제목 | 진행률 | 마지막 작업 |
|------|--------|-------------|
| 트랜스포머 아키텍처 | 2/5 | 2일 전 |

## ⏸️ 일시중지
| 제목 | 진행률 | 멈춘 사유 |
|------|--------|-----------|
| RAG 시스템 | 3/4 | "주말에 다시" |

## ✅ 완료 (최근 N개)
| 제목 | 완료일 |
|------|--------|
| Rust ownership 기초 | 2026-04-10 |

## 다음 액션 제안
- 트랜스포머 아키텍처 작업 계속 → "이어서 할게" 또는 `/study-goal:resume`
- RAG 시스템 재개 → `/study-goal:resume rag-systems`
- 새 목표 시작 → `/study-goal:new` 또는 그냥 "X 공부하고 싶어"
- 전체 명령어 다시 보기 → "명령어 알려줘"

뭐 하실래요?
```

표가 너무 길면 (각 그룹 5개 이상) 최근 순으로 잘라서 표시 + "더 보기는 `/study-goal:list`".

### 3. 명령어 도움말 재요청 처리

기존 사용자가 "명령어 알려줘", "사용법 다시 보여줘" 같이 말하면
§2-A의 명령어 표 부분만 다시 보여줌 (환영 인사 생략).

## 자연어 트리거

다음 표현은 이 명령으로 매핑:
- "study-goal 사용법", "어떻게 써?", "튜토리얼"
- 처음 설치 후 첫 메시지로 "안녕"이나 "시작" 같은 모호한 인사
