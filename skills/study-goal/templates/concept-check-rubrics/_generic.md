# 루브릭 메타 가이드 (Lazy Generation 가이드)

이 파일은 **루브릭이 아니다**. Claude가 새로운 토픽의 루브릭을
즉석에서 생성할 때 따라야 할 규칙과 형식을 담은 메타 가이드.

매칭되는 루브릭 파일이 없을 때 Claude는 이 가이드를 읽고
`~/study-log/.claude-study-goal/rubrics/<slug>.md`에 새 루브릭을 만든다.

---

## 1. 5단계 난이도의 의미 (토픽 무관 공통)

### L1 전제 (Prerequisite)
이 토픽을 시작하려면 미리 알아야 하는 **더 기초적인** 개념.
이게 없으면 진도 자체가 안 나감.
- 트랜스포머의 L1 → 행렬 곱셈, 신경망 forward/backward
- React의 L1 → JavaScript ES6 문법, DOM 개념
- 그래프 알고리즘의 L1 → 자료구조(배열/리스트), 재귀

### L2 기초 (Fundamentals)
이 토픽 입문자가 **가장 먼저 배우는** 용어와 개념.
교과서/튜토리얼 1-2장에 나오는 것들.

### L3 핵심 (Core)
이 토픽의 **정체성**을 만드는 중심 개념.
"이걸 모르면 이 토픽을 안다고 할 수 없는" 것들.
첫 질문은 항상 L3에서 시작한다.

### L4 응용 (Applied)
L3를 실제 문제에 적용할 때 필요한 추가 지식.
"왜", "언제", "어떤 상황에서"의 질문에 답할 수 있는 수준.

### L5 심화 (Advanced)
전문가 수준. 최적화, 변형, 한계, 최신 동향, 디버깅 직관.
이 단계 질문에 답할 수 있으면 강의 가능 수준.

---

## 2. 좋은 루브릭의 체크리스트

새 루브릭을 만들 때 다음을 모두 만족시켜라:

- [ ] 각 레벨에 핵심 개념 **2-4개** (1개 너무 적음, 5개 이상 너무 많음)
- [ ] 각 레벨에 샘플 질문 **1-2개**
- [ ] 샘플 질문은 **서술형** (Yes/No나 4지선다 X)
- [ ] 샘플 질문은 **암기 확인이 아니라 이해 확인**
  - 나쁨: "트랜스포머는 몇 년에 발표됐나요?" (날짜 암기)
  - 좋음: "왜 attention이 RNN을 대체할 수 있었나요?" (이해 확인)
- [ ] L1과 L5의 거리감이 충분히 큼
- [ ] 인접한 레벨 간 단계가 자연스러움 (L2 → L3에서 갑자기 점프 X)

## 3. 분야별 차원 가이드

토픽의 분야에 따라 어떤 "차원"으로 5단계를 나눌지 다르다.

### ML/DL/AI
- 차원: **수학적 직관** + **구현** + **학습 동역학**
- L1-L2는 수학/통계 전제, L3는 아키텍처 핵심, L4는 학습 trade-off, L5는 최신 변형/한계
- 예시 토픽: 트랜스포머, RAG, RLHF, diffusion model

### 프로그래밍 언어
- 차원: **문법** + **메모리/실행 모델** + **생태계/관용구**
- L1은 다른 언어 경험 (있으면), L2는 기본 문법, L3는 그 언어의 특이한 모델 (Rust면 ownership), L4는 실전 패턴, L5는 unsafe/내부 동작
- 예시 토픽: Rust, Go, Haskell, Zig

### 알고리즘/자료구조
- 차원: **자료구조** + **시간/공간 복잡도** + **응용**
- L1은 기초 자료구조, L2는 알고리즘 기본, L3는 핵심 알고리즘 패턴, L4는 응용/최적화, L5는 고급 자료구조
- 예시 토픽: 그래프 알고리즘, DP, 세그먼트 트리

### 프론트엔드/웹
- 차원: **렌더링 모델** + **상태 관리** + **생태계**
- L1은 HTML/CSS/JS 기초, L2는 프레임워크 기본, L3는 그 프레임워크의 핵심 추상화, L4는 성능/패턴, L5는 내부 동작/SSR 등
- 예시 토픽: React, Vue, Next.js

### 시스템/인프라
- 차원: **개념** + **trade-off** + **운영 실전**
- L1은 OS/네트워크 기초, L2는 도구 기본 사용법, L3는 핵심 추상화, L4는 production trade-off, L5는 디버깅/장애 대응
- 예시 토픽: Kubernetes, Kafka, PostgreSQL

### 도메인 지식 (역사/경제/철학 등)
- 차원: **사실** + **인과관계** + **해석/논쟁**
- L1은 시대/배경, L2는 주요 인물/사건, L3는 핵심 인과관계, L4는 다양한 해석, L5는 학계 논쟁/사료 비판
- 예시 토픽: 산업혁명, 금융위기, 칸트 철학

### 토픽이 위 분류에 안 맞으면
"이 토픽을 처음 가르치는 사람이라면 어떤 순서로 설명할까?"를 상상하고
그 순서를 5단계로 매핑.

## 4. Lazy Generation 절차

새 토픽에 대해 Claude가 따라야 할 순서:

1. **토픽 분석** — 분야 추론, 하위 영역 파악
2. **차원 선택** — 위 가이드에서 적절한 차원 선택
3. **5단계 후보 작성** — 각 단계에 무엇이 들어갈지 brainstorm
4. **각 단계 핵심 개념 2-4개 추출** — 너무 많으면 줄이기
5. **샘플 질문 1-2개씩 작성** — 위 체크리스트 따라
6. **검토** — 인접 단계 간 점프가 자연스러운지, L1-L5 거리가 충분한지
7. **저장** — `~/study-log/.claude-study-goal/rubrics/<slug>.md`로

**이 과정을 사용자에게 보여주지 않음.** 백그라운드에서 진행하고
"루브릭 준비 완료, Step 2 시작할게요"로 넘어감.

## 5. 루브릭 파일 형식

생성된 루브릭은 다음 형식을 따라야 한다:

```markdown
# Topic: <토픽 이름>
# Aliases: <alias1>, <alias2>, <alias3>
# Generated: <YYYY-MM-DD>
# Mode: <topic 또는 link>

## L1 전제 (Prerequisite)
- 개념 1
- 개념 2

샘플 질문:
- "..."

## L2 기초 (Fundamentals)
...

## L3 핵심 (Core)
...

## L4 응용 (Applied)
...

## L5 심화 (Advanced)
...

## Notes
<선택사항: 이 토픽 특유의 주의사항이나 단계 간 점프 포인트>
```

`Aliases`에는 검색 매칭에 쓸 키워드들. 한국어/영어/약어 다 포함.
예: `Aliases: 트랜스포머, transformer, attention, gpt`

## 6. 링크 모드의 차이

링크 모드(`Mode: link`)에서는 루브릭이 **콘텐츠 특화**가 된다.

- L1은 **이 콘텐츠를 읽기 위한 전제**
- L2-L4는 **이 콘텐츠 안의 개념들**을 난이도 순으로
- L5는 **이 콘텐츠가 다루지 않는 확장 주제**

콘텐츠 분석 결과(Step 1.5)가 입력으로 들어오므로, 그 안의 키워드와
구성 요소를 참고해서 루브릭을 만든다.

링크 모드 루브릭은 1회용에 가깝지만, 같은 콘텐츠를 다시 공부하거나
비슷한 콘텐츠 학습 시 참고가 될 수 있으므로 저장한다.

---

## 7. 학습 종료 후 다듬기 (옵션)

학습이 완료(모든 체크 항목 `[x]`)되면, 그 학습의 실제 진행을 토대로
루브릭을 개선할 수 있다:

- 어떤 개념이 실제로 핵심이었는지 (L3에 추가)
- 어떤 개념이 의외로 어려웠는지 (난이도 조정)
- 어떤 샘플 질문이 잘 작동했는지 (다음 학습자를 위해 교체)

업데이트 시 파일 상단에 `Last refined: <date>` 기록.
