study-goal 스킬의 일시중지 흐름을 실행한다.

사용자 입력: $ARGUMENTS

## 인자 파싱

$ARGUMENTS의 첫 단어를 slug로 간주, 나머지를 사유로 간주.
slug가 없으면 활성 목표 중에서 선택받기.

## 절차

1. 대상 slug 결정:
   - 인자에 slug 있음 → 사용
   - 인자 없음 + `.active`에 1개 → 그 목표 사용
   - 인자 없음 + `.active`에 여러 개 → AskUserQuestion으로 선택받기
   - 인자 없음 + `.active` 비어있음 → "일시중지할 활성 목표가 없어요"

2. `~/study-log/<slug>/artifact.md` 확인
   - 없으면: 에러 + 비슷한 slug 제안
   - `🟢 완료` 상태면: "이 목표는 이미 완료됐어요. 일시중지가 의미 없습니다"

3. artifact.md 수정:
   - 상태를 `🟡 진행 중` → `🔴 일시중지`로 변경
   - 작업 로그 섹션에 일시중지 기록 append:
     ```
     ### YYYY-MM-DD HH:MM
     **일시중지**: <사유 또는 "사유 미기재">
     **현재 진행률**: M/N 항목 완료
     **재개 시 다음 스텝**: <마지막 "다음 스텝"이 있다면 그것을 복사>
     ```

4. `.active`에서 해당 slug 제거
   - 다른 slug가 남아있으면 그대로 multi-active 유지
   - 마지막이었으면 `.active` 빈 파일로

5. 사용자에게 확인:
   - "✋ <제목> 일시중지 완료. `/study-goal:resume <slug>` 또는 자연어로 언제든 재개 가능"
   - 다른 활성 목표가 있으면: "지금 활성 목표는 [목록]이에요"

## 사유 안 적었을 때

사유는 강제하지 않음. 다만 인자에 사유가 없으면 한 번만 물어보기:
- "일시중지 사유를 짧게 적어두면 나중에 재개할 때 도움이 돼요. 한 줄만 알려주세요 (그냥 엔터쳐도 됨)"
- 빈 답변이면 "사유 미기재"로 기록하고 진행
