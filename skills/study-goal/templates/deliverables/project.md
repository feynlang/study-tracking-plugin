# 프로젝트형 산출물 변형 생성 가이드

## 4가지 변형 생성 패턴

목표 + 수준 + 종료조건을 바탕으로 다음 스코프 단계로 변형 4개:

1. **Tiny** — 1 모듈/1 파일, 몇 시간
2. **Small** — 작동하는 데모, 0.5~2일
3. **Medium** — 실제 사용 가능한 도구, 2~5일
4. **Stretch** — 시각화/UI/배포까지, 1주+

각 변형마다:
- 이름 (한 줄, 구체적)
- 무엇을 만드는지 (1-2문장)
- 체크가능한 완료 항목 3-5개
- 예상 소요 시간

수준에 따라 4개 중 추천을 다르게:
- 초급 → Tiny ~ Small
- 중급 → Small ~ Medium
- 상급 → Medium ~ Stretch

## 완료 체크 항목 작성 가이드

각 변형의 완료 항목에 다음을 포함:
- 특정 파일/엔트리포인트가 존재
- 명시적 입력 → 명시적 출력이 검증됨
- README 또는 사용법 1문단
- (선택) 테스트 1개 이상 통과
- (선택) 직접 실행해서 결과 확인
